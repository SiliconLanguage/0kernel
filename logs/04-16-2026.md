## Issues Encountered And Resolutions

### 1. Broken `index.html` (duplicate/old HTML appended)
- Symptom:
  - App rendered incorrectly / dark or inconsistent initial view.
  - Build warnings and stale script hints appeared.
- Root cause:
  - Legacy HTML content was appended after the closing `</html>` of the new Vite document.
- Resolution:
  - Rewrote `index.html` cleanly as a single valid document.
  - Revalidated with successful build.

### 2. Logo showing uppercase unexpectedly
- Symptom:
  - Top-left brand text appeared uppercase despite intended lowercase style.
- Root cause:
  - Inherited/competing styling from prior state and stale document context.
- Resolution:
  - Ensured clean HTML and added explicit style safeguards in navbar.

### 3. TypeScript build error on Worker env type
- Symptom:
  - `Cannot find name 'Fetcher'` in `src/index.ts`.
- Root cause:
  - `Fetcher` type not available in the current TS environment config.
- Resolution:
  - Replaced with a local `AssetFetcher` interface type and updated env typing.

### 4. Production favicon endpoint returning HTTP 500
- Symptom:
  - `https://0kernel.ai/favicon.svg` returned 500 while `/` returned 200.
- Root cause:
  - Runtime path handling and deployment mismatch during Worker+assets serving.
- Resolution:
  - Hardened Worker logic for asset binding edge-cases.
  - Added explicit fallback behavior for icon handling.

### 5. Cloudflare deploy failure due to invalid `_redirects`
- Symptom:
  - Wrangler deploy failed with `Invalid _redirects configuration` and infinite loop detection.
- Root cause:
  - Catch-all rule in `_redirects` conflicted with route processing.
- Resolution:
  - Removed problematic catch-all from redirect file used in deployment.
  - Kept minimal safe asset passthrough rule set.

### 6. SVG favicon worked, ICO still missing/failing on some clients
- Symptom:
  - `/favicon.svg` worked, but `/favicon.ico` failed or did not render in some browsers.
- Root cause:
  - Some clients request `.ico` explicitly; no reliable `.ico` available at edge.
- Resolution:
  - Generated and added `public/favicon.ico`.
  - Added explicit icon links in `index.html`.
  - Redeployed and verified both endpoints.

### 7. Local works but production still stale
- Symptom:
  - Local build looked correct while production still served older head markup/cached behavior.
- Root cause:
  - Deployment lag/caching and route/runtime version mismatch.
- Resolution:
  - Performed direct `wrangler deploy` and revalidated live endpoints.
